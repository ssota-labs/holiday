# 대한민국 사회보험 참고 (2026)

국민연금·건강보험·장기요양·고용·산재를 다룰 때 읽는다. **요율·상하한은
추측하지 않는다.** 직장 급여 공제는 CLI `KR_STATUTE`(기준일 `2026-01-01`)가
강제하고, 이 파일이 그 숫자·자격 개념을 정리한다. 법이 바뀌면
`packages/core/src/domain/kr-income.ts`와 **이 문서를 같은 PR에서** 올린다.

수입 정산 명령·regime은 [`kr-income.md`](kr-income.md)를 본다.

## 언제

- “4대보험이 뭐가 빠지나 / 요율이 얼마냐”를 물을 때
- 직장가입·지역가입·임의가입 등 **자격**을 설명할 때
- 급여 명세·공단 고지서 숫자를 장부에 옮길 때
- 지역가입·직접 납부처럼 CLI가 계산하지 않는 금액을 다룰 때

## 하지 말 것

- 머리로 요율을 곱해 `txn add` leg를 만들지 않는다.
- 지역가입·임의가입·추납 금액을 직장 요율로 **추정**하지 않는다 — 고지서·납부확인서 관측값만.
- 산재·고용안정사업 사업주 분까지 급여 공제에 넣지 않는다 (근로자 명세에 없음).
- 이 파일을 “보험료 자동 계산기”로 쓰지 않는다. 참고표다.

## 구성 (흔히 말하는 4대 + 장기요양)

| 이름 | 역할 | 근로자 급여에서 |
|---|---|---|
| 국민연금 | 노령·장애·유족 연금 | 근로자분 공제 |
| 건강보험 | 의료보장 | 근로자분 공제 |
| 장기요양보험 | 요양 급여 (건보에 연동) | 근로자분 공제 (건보료 × 비율) |
| 고용보험 | 실업급여 등 | 실업급여 근로자분만 공제 |
| 산재보험 | 업무상 재해 | **공제 없음** (사업주 전액) |

일상에서 “4대보험”은 연금·건보·고용·산재를 말하고, 명세에는 장기요양이
건보 옆에 따로 찍히는 경우가 많다. CLI `salary` regime 라인은
국민연금·건강보험·장기요양·고용보험(실업)이다.

## 자격 (누가 내는가)

| 자격 | 대략 | 보험료 |
|---|---|---|
| 직장가입자 | 사업장에 고용 | 근로자·사용자 분담. 개인 고지 없음(급여 공제) |
| 지역가입자 | 직장 밖 (거주·소득 기준) | **본인이 고지액 납부** |
| 임의가입 | 국민연금만 — 의무 가입이 아닐 때 선택 | **본인 납부** |

자격이 바뀌면(퇴사·입사·취득·상실) 고지 유무도 바뀐다. 지금은 직장이라
고지가 없어도, 나중에 지역으로 바뀌면 직접 낸다. 자격 이력·월 고지를
장부 SoR로 두는 기획은 docs `prd-social-insurance-sor`를 본다 (구현 전이면
전표·메모로만).

## 2026 요율 — 직장가입자 (CLI와 동기)

기준일 `2026-01-01` (`KR_STATUTE_AS_OF`). 아래 **근로자 부담**이
`holiday income settle --regime salary`가 계산하는 값이다.

| 항목 | 전체 | 근로자 | 사용자 | CLI |
|---|---|---|---|---|
| 국민연금 | 9.5% | 4.75% | 4.75% | ✓ `national_pension` |
| 건강보험 | 7.19% | 3.595% | 3.595% | ✓ `health_insurance` |
| 장기요양 | 건보료의 13.14% (소득 대비 ≈0.9448%) | 근로자 건보료 × 13.14% | 동일 | ✓ `long_term_care` |
| 고용보험(실업급여) | 1.8% | 0.9% | 0.9% + 규모별 고용안정·직능 | ✓ 실업 0.9%만 |
| 산재보험 | 업종별 | 없음 | 전액 | ✗ 미취급 |

절사: 원 미만 절사 (CLI `truncApplyRate`).

### 국민연금 기준소득월액 (상·하한)

보수월액이 아니라 **기준소득월액**에 요율을 곱한다. 천 원 미만 절사 후 상·하한.

| 적용 기간 | 하한 | 상한 |
|---|---|---|
| ~2026-06-30 | 400,000원 | 6,370,000원 |
| 2026-07-01~ | 410,000원 | 6,590,000원 |

상한을 넘는 급여여도 연금 보험료는 상한 기준이다.

### 장기요양 계산 순서 (직장)

1. 건강보험료(근로자분) = 보수월액 × 3.595% (원 미만 절사)
2. 장기요양(근로자분) = ① × 13.14% (원 미만 절사)

월급에 0.9448%를 직접 곱하는 방식과 결과가 같도록 고시된 쌍이다.
CLI는 **건보료 × 13.14%** 경로를 쓴다.

### 고용보험 — 근로자에게 보이는 것만

- 명세 공제: 실업급여 **0.9%**
- 사업주 추가(고용안정·직업능력개발, 사업장 규모 0.25~0.85%p 등)는
  급여에서 빠지지 않는다. CLI도 넣지 않는다.

## 지역가입·직접 납부

지역 건보·연금·임의가입·추납 금액은 **공단 고지·납부확인서 숫자만** 옮긴다.
직장 요율표로 역산하지 않는다.

- 통장에서 나갔으면 → 보험 비용·납부 전표
- “그달 고지액이 얼마였나”를 남기려면 → (기획 승인 후) 사회보험 월 고지 SoR;
  그 전에는 전표 narration·메모로만

갑근세·지방소득세는 4대보험이 아니다. 급여 regime에서는 `--earned-tax`로
명세서/간이세액표 값을 넣는다 (`kr-income.md`).

## CLI와의 경계

| 질문 | 어디 |
|---|---|
| 이번 급여에서 4대보험 근로자분이 얼마인가 | `income settle --regime salary` |
| 요율·상하한이 법령과 맞나 | `income check` + 이 표 / `KR_STATUTE` |
| 사업소득 3.3%·부가세 10% | `kr-income.md` (사회보험 아님) |
| 지역가입 고지액 | 관측값만. 요율 추정 금지 |
