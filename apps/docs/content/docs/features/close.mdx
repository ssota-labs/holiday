## 개요

월 마감은 회계 기간을 확정하는 절차입니다. CLI의 hard-close 단위는 **month**
(`YYYY-MM`)입니다. book의 `closeGrain`이 `month`가 아니면 `close`는 거부됩니다.

다음 두 가지는 의도적으로 수행하지 않습니다 (`close.ts`).

1. **다중 단위 hard close 없음** — 일과 달을 함께 재평가하면 FX가 이중 계상됩니다. hard close 단위는 book당 하나입니다.
2. **자본(equity) 결산 분개 없음** — 기간 합계는 SQL로 산출합니다. 이월·대차대조표는 스냅샷으로 보존합니다.

일·주 단위는 soft checkpoint(스냅샷만)입니다. hard close의 재평가와
`period.status=closed`는 적용하지 않습니다.

## 동작

### 잔액 대사 (`assert`)

`holiday assert <account> <amount>`는 명세서상 잔액을 기록하고, 동일 기준일의
장부 잔액과 대조합니다. 불일치 시 exit 1(`assertion_failed`)이며, 대사 기록은
저장됩니다.

부채·자본·수익은 표시 부호를 보정합니다(장부 크레딧 잔액 vs 명세서 잔액).

잔액 대사는 원장과 외부 자료를 비교하는 유일한 검사입니다. 마감 게이트는 기간 내
대사 결과를 다시 검사합니다.

### closeGate

`close <month>`는 먼저 `closeGate`를 통과해야 합니다.

| 조건 | 실패 시 |
|---|---|
| 기간 내 `status='draft'` 거래 수 > 0 | 마감 차단. `review`로 처리 |
| 기간 내 잔액 대사(`balance_assertion`) 중 불일치 | 마감 차단. 차액 출력 |

통과 후 재평가 → 스냅샷 → `period.status = closed` 순으로 진행합니다.
재평가는 기간이 아직 open인 동안 기표합니다.

### FX 재평가

월말 잔고 중 **화폐성 자산·부채**이며 기능통화가 아닌 계정을 재평가합니다.

- `--non-monetary` 계정 제외
- 비용·수익·자본 제외(거래일 측정)
- `units = 0`이어도 carrying weight가 남으면 재평가 대상(실현 FX)
- posting: `units_minor = 0`, `weight_minor = delta`, 상대 계정 `Income:FX:Unrealized`

재평가 기표 후 스냅샷을 기록하고 `period.status`를 `closed`로 변경합니다.

### --dry-run

게이트·재평가 예정만 출력하고 쓰지 않습니다.

잔액 대사가 0건이어도 마감은 가능합니다. 다만 CLI는 명세서 대조가 없는 달에
대해 경고합니다.

## 제약

- hard close 입력은 `YYYY-MM`뿐입니다. book `closeGrain`이 `month`여야 합니다.
- draft 또는 실패한 잔액 대사가 있으면 마감하지 않습니다.
- equity closing entry를 만들지 않습니다.
- 재평가에 `Income:FX:Unrealized` 계정이 필요합니다.
- 환율이 없으면 외화 잔고 재평가가 실패하여 마감이 중단됩니다. 사전에 `fx add`가 필요합니다.
- `period.status = closed`는 기록됩니다. 닫힌 기간 날짜로의 `txn add` 거부는 아직 store에 없습니다.
