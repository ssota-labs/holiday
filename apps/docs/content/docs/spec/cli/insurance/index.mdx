## Schemes / enrollment status

| `--scheme` | 의미 | 허용 `--status` |
|---|---|---|
| `health` | 건강보험 (장기요양 자격은 건보에 종속) | `workplace` · `regional` |
| `national_pension` | 국민연금 | `workplace` · `regional` · `voluntary` |

| `--status` | 의미 |
|---|---|
| `workplace` | 직장가입자 — 보통 개인 고지 없음 |
| `regional` | 지역가입자 — 직접 납부 |
| `voluntary` | 임의가입 (국민연금만) — 직접 납부 |

## Contribution kinds

| line `kind` | 의미 |
|---|---|
| `national_pension` | 국민연금 |
| `health_insurance` | 건강보험 |
| `long_term_care` | 장기요양 |

## insurance enrollment add

<CommandSpec
  command="insurance enrollment add"
  mutates={true}
  summary="사회보험 자격 구간을 append한다."
  signature={`holiday insurance enrollment add --scheme --status --from [--to] [--note]`}
  flags={[
    { name: '--scheme', type: 'health|national_pension', required: true, description: '보험 종류' },
    { name: '--status', type: 'workplace|regional|voluntary', required: true, description: '자격' },
    { name: '--from', type: 'YYYY-MM-DD', required: true, description: '시작일' },
    { name: '--to', type: 'YYYY-MM-DD', description: '종료일. 생략 시 현재까지(열린 구간)' },
    { name: '--note', type: 'string' },
  ]}
  exits={[
    { code: 0, meaning: '성공' },
    { code: 1, meaning: 'overlap · invalid_status_for_scheme · LedgerError' },
    { code: 2, meaning: '사용법 오류' },
  ]}
  example={`holiday insurance enrollment add --scheme health --status workplace --from 2025-09-01 --json`}
/>

같은 `scheme`에서 날짜가 겹치는 열린/닫힌 구간과 충돌하면 거절한다.
같은 `scheme`에 **열린 구간이 하나** 있고 새 구간의 `--from`이 그보다 뒤면,
이전 구간을 새 시작일 **전날**로 자동 종료한 뒤 append한다. 열린 구간이
둘 이상이면 거절한다 — 그 경우 이전 구간에 `--to`를 명시한 뒤 다시 add.

### `--data-file`

```json
{
  "commodity": "KRW",
  "lines": [
    { "kind": "health_insurance", "amount": "143800" },
    { "kind": "long_term_care", "amount": "18890" },
    { "kind": "national_pension", "amount": "189000" }
  ]
}
```

금액은 **소수 문자열**. `number` 금지. `lines` ≥ 1, kind 중복 금지.
직장 공제액을 넣지 않는다 — 직접 납부 달만.
