## 한 줄로

개인사업자가 **지금 어떤 사회보험 자격인지**(직장가입·지역가입 등)를 기간으로
남기고, 자격이 **직접 납부**로 바뀌는 달에는 국민연금·건강보험 등 금액을
세금처럼 **월별로** 모아 본다.

## 문제

1인 사업자는 자격이 자주 바뀐다. 예:

- 지금은 **직장가입자**라 공단 고지가 안 온다 (회사에서 공제)
- 퇴사·휴직·사업만 남으면 **지역가입자**로 바뀌고 그때부터 본인이 낸다
- 국민연금은 직장·지역·임의가입처럼 자격이 갈린다

지금 장부에는:

1. **급여 정산** — 직장 공제분(이번 급여에서 얼마가 빠졌나)만 맞춘다
2. **통장 전표** — 직접 이체한 보험료는 비용으로만 잡힌다

빠진 것은 두 가지다.

- **자격 이력** — “언제부터 언제까지 직장/지역이었나”
- **직접 납부 달의 금액표** — “지역가입이던 2025년 4~8월 건보·연금이 얼마였나”

세금은 신고서를 기간별로 따로 둔다. 사회보험에는 **자격 구간**과
**내가 내야 하는 달의 고지액**을 둘 자리가 없다. 공단 화면·엑셀을 다시 연다.

## 무엇을 저장하는가

저장 층은 둘이다. 자격과 금액은 질문이 다르다.

### 1) 자격 구간

“그 보험에서 내가 어떤 가입자였는지”를 날짜 구간으로 남긴다.

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 보험 종류 | 건강보험 / 국민연금 | 자격 축이 보험마다 다르다 |
| 자격 | 직장가입자 / 지역가입자 / 임의가입 | “누가 보험료를 부담하는가”의 기준 |
| 시작일 | 2025-09-01 | 이 날부터 이 자격 |
| 종료일 | (없음) 또는 2026-02-28 | 없으면 “현재까지”. 바뀌면 이전 구간을 닫는다 |
| 메모 | 선택 | 취득·상실 사유 짧게 |

1차에서 다루는 자격:

| 보험 | 자격 |
|---|---|
| 건강보험 (장기요양 포함 안내) | 직장가입자 · 지역가입자 |
| 국민연금 | 직장가입자 · 지역가입자 · 임의가입 |

같은 보험에서 **열린 구간이 겹치면** 거절한다. 자격이 바뀌면 이전 구간에
종료일을 넣고 새 구간을 연다.

직장가입자인 동안에는 보통 **개인 고지가 없다**. 그래도 자격 구간은 남긴다 —
나중에 “그때는 직장이었다”를 되묻지 않으려고.

### 2) 직접 납부 월 금액

자격이 직접 납부인 달(지역가입·임의가입 등)에, 고지서·납부확인서에 찍힌
금액을 월 단위로 옮긴다. 요율은 다시 적용하지 않는다.

| 사람이 읽는 이름 | 예시 | 왜 필요한가 |
|---|---|---|
| 부과연월 | 2025-05 | 몇 월분인지 |
| 기록일 | 2025-05-20 | 옮겨 적은 날 |
| 기록 차수 | 1차, 2차(정정)… | 금액을 고쳐도 이력 |
| 국민연금 | 189,000 | 그달 고지·확정액 |
| 건강보험 | 143,800 | |
| 장기요양 | 18,890 | 건보와 같이 오는 경우가 많음 |

빈 칸은 저장하지 않는다. **항목이 하나도 없으면** 거절한다.

금액 층은 **직접 납부**만 1차 범위다. 직장 공제액은 급여 정산·명세 쪽에 두고
이 층에 중복으로 넣지 않는다(아래 Non-goals).

### 3) 정정은 덮어쓰지 않는다

월 금액을 고칠 때는 이전 기록을 지우지 않고 **새 차수를 추가**한다.
이전 차수는 “대체됨”으로 남긴다.

### 4) 통장 납부·급여 정산과는 별개

자격·고지 기록을 넣는 동작은 **전표를 만들지 않는다**.
실제로 통장에서 낼 때는 지금처럼 **비용·납부 전표**로 따로 한다.
직장에서 빠진 분은 **수입 정산**이다.

| 질문 | 어디에 답이 있나 |
|---|---|
| 지금/과거에 직장·지역 중 무엇이었나 | 자격 구간 (이 PRD) |
| 지역가입이던 달 고지액이 얼마였나 | 직접 납부 월 금액 (이 PRD) |
| 통장에서 보험료로 얼마가 나갔나 | 전표·보험 비용 계정 |
| 이번 급여에서 4대보험이 얼마 빠졌나 | 수입 정산 (기존) |

자격과 월 금액을 자동으로 맞추지 않고, 자격만으로 금액을 만들지도 않는다.
1차는 사실을 남기고 다시 보는 데까지다.

## 목표

- 건강보험·국민연금 **자격 구간**을 저장·조회한다
- 직접 납부 달에 **월별 고지 금액**을 저장·조회한다
- 등록·목록으로 “언제 어떤 자격이었는지 / 그달 얼마였는지”를 다시 본다
- 월 금액 정정은 이력을 남긴다
- **요율로 금액을 추정하지 않는다** — 고지서·납부확인서 숫자만

## Non-goals

- 공단 API·PDF OCR로 자격·고지 자동 수집
- 요율표로 지역가입 보험료를 다시 계산
- **직장 공제액**을 이 SoR에 다시 넣는 것 (급여 정산이 담당)
- 산재·고용보험 자격 전용 화면, 퇴직연금·실손 등 법정 사회보험 밖 상품
- 자격 변경이 전표·스케줄을 자동 생성
- 웹 UI·다인 클라우드 장부
- 세금 신고서 테이블에 사회보험을 섞기

## 비기능

- 금액은 원 단위 정수
- 이미 적용된 DB 변경은 고치지 않고 새 변경만 추가한다
- 기존 장부 의존 방향을 지킨다

## 성공 지표

1. 건강보험 자격으로 “2025-09-01부터 직장가입자(종료 없음)”를 넣으면, 조회에
   그 구간이 보인다
2. 같은 보험에 “2026-03-01부터 지역가입자”를 넣으면, 이전 직장 구간에 종료일이
   생기고 새 구간이 열린다 (또는 종료일을 명시한 뒤 새 구간을 연다)
3. 지역가입 기간의 2026-03분 월 금액(건보·장기요양 등)을 넣으면 고지서와 같은
   숫자로 조회된다
4. 같은 부과연월·같은 차수 월 금액을 다시 넣으면 **거절**된다
5. 자격·월 금액을 넣어도 **전표는 생기지 않는다**
